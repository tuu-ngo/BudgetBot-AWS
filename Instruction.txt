2. THAY ĐỔI DATABASE SCHEMA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Bảng transaction — thêm 2 cột:
  confident     NUMERIC(4,2)  — đổi từ INTEGER sang float 0.00–1.00
  review_status VARCHAR(20)   — 'ok' | 'review' (confidence < threshold)

Bảng file — thêm 1 cột:
  status        VARCHAR(20)   — 'pending' | 'processing' | 'done' | 'error'
                                (dùng cho aws flow khi parse async qua SQS)

Bảng budget_cap — MỚI (Table 5):
  cap_id      UUID PK
  user_id     UUID FK → user
  category    category_enum
  cap_amount  BIGINT         — ngưỡng chi tiêu (VND)
  UNIQUE(user_id, category)  — mỗi user chỉ có 1 cap / category

Migration idempotent — an toàn khi chạy lại trên DB đã có dữ liệu:
  ALTER TABLE ... ADD COLUMN IF NOT EXISTS ...
  ALTER TYPE category_enum ADD VALUE IF NOT EXISTS ...

4. FLOW CHI TIẾT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

── FLOW_MODE=local (dùng để test ở máy) ──────────────────────────────────────

POST /upload
  → api_handler.handle_upload()
  → lưu file vào LocalStorage (./_data/uploads/)
  → tạo file record trong RDS (status='done')
  → gọi parser_handler.process_rows() INLINE (không cần SQS)
      → với mỗi row CSV:
          → ai_client.categorize(description, amount, date)
          → trả về {category, confidence: 0.0–1.0}
          → confidence >= 0.60 → review_status='ok'
          → confidence <  0.60 → review_status='review'
      → INSERT INTO transaction
  → gọi budget_handler.check_and_alert() INLINE
      → query budget_cap table
      → query spending_this_month()
      → nếu vượt cap → SNS.publish() (bỏ qua nếu SNS_TOPIC_ARN trống)
  → return JSON kết quả (rows_inserted, rows_review, sample_categorized)

── FLOW_MODE=aws (trên Lambda thật) ──────────────────────────────────────────

POST /upload  [Lambda 1 — API Lambda]
  → tạo file record trong RDS (status='pending')
  → generate S3 presigned PUT URL (expires 5 phút)
  → gửi SQS message {user_id, file_id, s3_key, filename}
  → return {presigned_url, file_id, s3_key, status: 'pending'}

  [User PUT file lên S3 qua presigned_url]

SQS trigger → Lambda 2 — Parser Lambda
  → update file status → 'processing'
  → storage.get(s3_key) — download CSV từ S3
  → parse_csv() → rows
  → với mỗi row: BedrockAI.categorize() few-shot (confidence 0.0–1.0)
  → INSERT INTO transaction (review_status theo threshold)
  → update file status → 'done'
  → boto3.invoke(BUDGET_LAMBDA_NAME, InvocationType='Event')  ← async, non-blocking

Lambda 4 — Budget Handler  [triggered async]
  → query budget_cap table
  → query spending_this_month() từ RDS
  → với mỗi category vượt cap: SNS.publish() → email alert

GET /upload/{file_id}/status  — poll trạng thái xử lý file

5. API ENDPOINTS (đầy đủ)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

GET    /health                          Kiểm tra trạng thái + backends
POST   /users                           Tạo user mới
GET    /users/{user_id}                 Lấy thông tin user
PUT    /users/{user_id}                 Cập nhật user
DELETE /users/{user_id}                 Xoá user + cascade

POST   /upload                          Upload bank statement CSV
GET    /upload/{file_id}/status         Poll trạng thái xử lý file (aws mode)

GET    /summary?month=YYYY-MM           Tổng hợp chi tiêu theo category
GET    /transactions?month=&review_status=  Danh sách transactions (filter được)
GET    /transactions/review             Chỉ các transaction cần review (confidence < 60%)

GET    /files                           Danh sách file đã upload

GET    /budget-caps                     Danh sách budget caps của user
PUT    /budget-caps/{category}          Set/update cap (body: {cap_amount: int})
DELETE /budget-caps/{category}          Xoá cap

POST   /chat                            Chat với AI (body: {message, month?})
GET    /chat/history?limit=50           Lịch sử chat

POST   /budget/check                    Trigger kiểm tra budget thủ công (test local)

GET    /                                Frontend HTML (nếu SERVE_FRONTEND=true)

Header xác thực: X-User-Id: <user_id>  (tất cả endpoints)

Few-shot examples (10 ví dụ cụ thể trong prompt):
  HIGHLANDS COFFEE BUI VIEN → Food,          confidence: 0.95
  SALARY DEPOSIT CREDIT     → Income,        confidence: 0.98
  T1908 GRAB CITY           → Transport,     confidence: 0.82
  NETFLIX SUBSCRIPTION      → Subscriptions, confidence: 0.97
  VINMART HCM 04            → Shopping,      confidence: 0.65
  EVN TPHCM TIEN DIEN       → Utilities,     confidence: 0.96
  FT0024112501 ID:0001      → Other,         confidence: 0.12  ← triggers review
  VIETTEL INTERNET          → Bills,         confidence: 0.94
  CGV CINEMA VINCOM         → Entertainment, confidence: 0.93
  LONG CHAU PHARMACY        → Health,        confidence: 0.91

Numeric confidence (thay "high|medium|low"):
  Bedrock trả về float 0.0–1.0 thay vì string
  Ngưỡng REVIEW_THRESHOLD=0.60 (configurable qua env)

BedrockAI.chat():
  - System prompt inject spending summary từ RDS
  - Multi-turn conversation history (last 10 turns)
  - maxTokens=512, temperature=0.7
  - Trả lời tiếng Việt hoặc tiếng Anh theo ngôn ngữ user dùng

LocalAI.chat():
  - Không gọi Bedrock
  - Trả về spending summary dạng text (dùng cho dev/test)
